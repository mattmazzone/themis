Guide de Style

Cette section de la documentation explique comment bien rédiger un PM pour qu'il soit bien reconnu par l'outil de conversion. Les fonctionnalités des sections seront également abordées

Insérer une checkbox

Pour insérer une checkbox dans la version Html du PM, il faut mettre des paranthèses au début de la phrase

Contenu PM.txt

( ) COMMUNIQUER AU CLIENT LES TERES DE CETTE INSPECTION.

( ) COORDONNER AVEC UN AUTRE MÉTIER SI NÉCESSAIRE.

Résultat PM.html


Les Sections d'un PM

Un PM contient plusieurs sections et devrait être structuré de la façon suivante:

Contenu PM.txt

<< PRÉPARATION>>

<< DOCUMENT RÉFÉRENCE>>

<< CONSOMMABLES>>

<< PIÈCES>>

<< OUTILS>>

<< SÉCURITÉ>>

<< TÂCHES>>

-FUNC_LOCATION:PWC-TFE-0001-TC10
-EQIP_NUMBER:M98110-10
-TASK_NUMBER:T98110-10MI2
-CADENASSAGE_LINK:https://www.google.ca

Les sections que vous voyez à gauche sont les plus communes parmi les PM. Alors, ils ont des filtres spécifiques dont on parlera en détails plus loin dans cette section. Veuillez notez qu'il est important de ne pas faire des fautes d'ortographes dans les titres des sections.

Si vous mettez un autre mot entre crochets, il serra reconnu quand même par le filtre "Unknowns".

Section PM: Préparation

La section Préparation du PM ajoute automatiquement des checkboxes. Ces checkboxes doivent obligatoirement être côchés pour soumettre le PM.

Fiche de communication

Cette section contient aussi le lien pour télécharger la fiche de communication(.xlsx). Le lien est statique et peut uniquement être changé dans le fichier htmlManip.cpp de l'outil de conversion.
Le lien est : O:/TFM_GESTION/PM/1_TEMPLATES/fiche_communication_ects.xlsx

Si la fiche de communication doit être changé, il faut renommer la nouvelle avec le nom de l'ancienne et la mettre exactement au même endroit.

Section PM: Documents Références

La section Documents Références ne peut pas contenir des inputs HTML.

Section PM: Consommables

La section Consommables ne peut pas contenir des inputs HTML.

Section PM: Pièces

La section Pièces ne peut pas contenir des inputs HTML.

Section PM: Outils

La section Outils ne peut pas contenir des inputs HTML.

Section PM: Sécurité

La section Sécurité, comme la section Préparation, ajoute automatiquement des checkboxes qui doivent être obligatoirement côchés pour la soumettre le PM.

Lien Cadenassage

Cette section contient aussi le lien de cadenassage. Ce lien doit être écrit dans le PM de la façon suivante:
-CADENASSAGE_LINK:www.somewebsite.com
C'est très important qu'il n'y a pas de fautes d'ortographes sinon le lien ne sera reconnu par l'outil de conversion.
Le PM.html affiche l'image de cadenassage qui est sur le drive O: au lien suivant: O:/TFM_GESTION/PM/1_TEMPLATES/LogoPratt/logo_cadenas.png
Si l'image de cadenassage doit être changé, il faut renommer la nouvelle avec le nom de l'ancienne et la mettre exactement au même endroit.

Section PM: Formulaire A3

La section Formulaire A3 est générée lors de la conversion d'un PM et non lors du chargement de la page web. Cela veut dire que s'il faut changer un élément dans le formulaire, il faut relancer l'outil de conversion pour mettre à jour le PM.
Pour modifier le formulaire il faut faire des changements dans le ficher à l'emplacement suivant:
O:/TFM_GESTION/PM/1_TEMPLATES/FormulaireA3/FormulaireA3.txt
Pour que la case "Oui" soit en rouge, il faut mettre un "Y" au début de la phrase dans le fichier FormulaireA3.txt
Pour que la case "Non" soit en rouge, il faut mettre un "N" au début de la phrase dans le fichier FormulaireA3.txt
Pour que aucune case soit en rouge, il faut pas mettre de "Y" ou "N" au début de la phrase dans le fichier FormulaireA3.txt
Voici un exemple:

Contenu FormulaireA3.txt

Questions
YEst-ce j'ai besoin de EPI spécifique?
YEst-ce que je dois cadenasser?
Est-ce qu'il y a d'autres sources d'énergie?
Est-ce que je dois me procurer un permis de travail?
YEst-ce que j'ai besoin d'aide?
NEst-ce que mes outils sont appropriés et conformes?

Résultat PM.html

Questions Oui Non

Section PM: Risques et Méthodes

Il n'y a rien à écrire dans le PM.txt pour la section risques et méthodes.

Section PM: Tâches

La section Tâches a beaucoup de fonctionnalités mais, elle est très intuitive a utiliser. Premièrement, il éxiste deux types de tâches. Une tâche principale et une tâche secondaire. On indique le type de tâche à l'outil de conversion avec le préfixe de la tâche. Une tâche secondaire est légèrement indentée pour la différencier de la tâche principale. Voici un exemple pour montrer comment spécifier si une tâche est principale ou secondaire.

Contenu PM.txt

( ) 1 - Tâche

( ) 1.a - Tâche

( ) 1.b - Tâche

( ) 2 - Tâche

( ) 3 - Tâche

( ) 3.a - Tâche

Résultat PM.html






Insérer un champ de texte

Pour ajouter un champ de texte, il faut ajouter: -INSERT_TEXTBOX à l'endroit qu'on le veut dans le PM.txt Aussi, si on veut ajouter un placeholder au champ de texte, on peut l'écrire après -INSERT_TEXTBOXthis is a placeholder! Voici un exemple qui montre une tâche avec un textbox ajouter:

( ) 1 - Tâche
-INSERT_TEXTBOXÉcrire le voltage de #1 ici!

( ) 1a - Tâche

( ) 1b - Tâche

Insérer une image

Il est possible d'ajouter des images a des tâches pour les expliquer. Il est déconseillé de mettre beaucoup d'image dans un PM parce que une page HTML avec beaucoup de lien externe va prendre beaucoup de temps à charger. Pour ajouter une image, il faut ajouter: -INSERT_PICTURE dans le PM.txt suivi par le lien de l'image. Par exemple, -INSERT_PICTURE

Contenu PM.txt

( ) 1 - Tâche

( ) 1.a - Tâche
-INSERT_PICTUREO:\TFM_GESTI...

( ) 1.b - Tâche

Résultat PM.html

Section PM: Ajouter des Images

Il n'y a rien à écrire dans le PM.txt pour la section infos soumission

C'est important de remarquer que les images qui seront upload de cette manère ne seront pas enregistré dans le template du PM. C'est vraiment juste pour que les techniciens puissent ajouter des photos à leur soumission.

Pour ajouter des photos au template il faut ajouter avec -INSERT_PICTURE dans le PM.txt

Section PM: Soumettre/Replanifier

Bouton Soumettre

Le bouton Soumettre est relié à une très grosse fonction JavaScript. Cette fonction vérifie les checkbox avec le class name "mustcheck", les radiobuttons du formulaire A3. Ensuite, elle vérifie si un checkbox "Avis Créé" est coché,l'attribut "required" est donné au champ de text respectif. Par la suite, la fonction vérifie si les cases Remplacé ou Réparé sont cochées. S'ils le sont, l'attribut "required" est donné au textarea respectif. Ensuite, les champs de text Infos Soumission sont aussi vérifié. Finalement, si tout est bien rempli, la fonction "window.print()" est appelée pour permettre à l'utilisateur d'enregistrer sa page en pdf. L'utilisateur devrait selectionner le dossier Téléchargment comme emplacement pour le pdf. Après l'utilisateur appuie sur enregistrer, la fenêtre d'impression va fermer et un téléchargement de "FileClient.hta" va commencer. On va en parler plus loin dans la documentation.

Bouton Replanifier

Le bouton Replanifier est relié a une fonction JavaScript qui vérifie uniquement si les champs de text Infos Soumission sont remplis. S'ils le sont, la fonction "window.print()" est appelée pour permettre à l'utilisateur d'enregistrer sa page en pdf. L'utilisateur devrait selectionner le dossier Téléchargment comme emplacement pour le pdf. Après que l'utilisateur appuie sur enregistrer, la fenêtre d'impression va fermer et un téléchargement de "EmailClient.hta" va commencer. On va en parler plus loin dans la documentation.
